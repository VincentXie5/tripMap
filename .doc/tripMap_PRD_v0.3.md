### 1. 版本目标

在v0.2完整CRUD功能闭环的基础上，增强地图与行程的交互体验，实现地图点位与行程的双向联动，并支持行程拖拽排序功能，提升行程编排的灵活性和直观性。

### 2. 版本范围

#### 2.1 必做核心功能

**地图点位与行程点击联动：**
- 点击地图点位标记，左侧对应行程项高亮显示并滚动至可视区域
- 点击左侧行程项，地图对应点位标记高亮并居中显示
- 点击空白区域取消所有高亮状态

**行程拖拽排序：**
- 每日行程列表支持拖拽调整顺序
- 拖拽后自动更新行程时间（可选：保持原时间或自动分配新时间）
- 拖拽过程中提供视觉反馈（占位符、半透明效果）

#### 2.2 本版本暂不实现功能

- 用户登录、注册、权限管理
- 响应式适配移动端、平板
- 行程导出、分享、路线规划
- 批量删除操作
- 操作历史记录与撤销功能

### 3. 页面结构设计

```Plain Text
首页（全屏布局）
├─ 左侧区域（固定占比40%）
│  ├─ 新建旅行计划表单
│  ├─ 旅行计划列表（日期区间 + 计划名 + [编辑] [删除]按钮）
│  └─ 选中计划的每日行程列表（时间：地点 + [编辑] [删除]按钮）
│      └─ 支持拖拽排序（拖拽手柄图标）
└─ 右侧区域（固定占比60%）
   └─ Leaflet地图容器（展示所有行程地点标记）
       └─ 点击点位触发联动高亮
```

### 4. 数据模型设计

#### 4.1 旅行计划实体（TravelPlan）

保持v0.2设计不变：
- id：主键ID，Long类型，自增
- title：计划名称，字符串类型
- startDate：旅行开始日期，日期类型
- endDate：旅行结束日期，日期类型

#### 4.2 每日行程实体（DailyPlan）

**新增字段：**
- sortOrder：排序序号，整数类型，用于记录行程在列表中的显示顺序

保持v0.2原有字段：
- id：主键ID，Long类型，自增
- planId：关联旅行计划ID，Long类型
- planDate：行程当天日期，日期类型
- time：行程具体时段，字符串类型（如：08:00、14:00）
- location：行程地点，字符串类型（如：北京故宫、西安兵马俑）

### 5. 核心接口清单

#### 5.1 v0.2已有接口（复用）

- POST /api/travelPlan：新建旅行计划
- GET /api/travelPlan：查询全部旅行计划列表
- PUT /api/travelPlan/{id}：更新旅行计划
- DELETE /api/travelPlan/{id}：删除旅行计划（级联删除关联行程）
- POST /api/dailyPlan：新增每日行程
- GET /api/dailyPlan/{planId}：根据计划ID查询对应全部行程
- PUT /api/dailyPlan/{id}：更新每日行程
- DELETE /api/dailyPlan/{id}：删除每日行程

#### 5.2 v0.3新增接口

**行程排序接口：**
- PUT /api/dailyPlan/sort/{planId}：批量更新行程排序
  - 请求体：[{id: 1, sortOrder: 1}, {id: 2, sortOrder: 2}, ...]
  - 响应：更新后的行程列表

### 6. 技术实现要点

#### 6.1 后端实现

- DailyPlan实体新增sortOrder字段，使用@Column注解定义默认值
- DailyPlanController新增updateSortOrder方法，接收排序数组
- DailyPlanService实现批量更新排序逻辑，使用@Transactional保证事务一致性
- DailyPlanRepository新增按planId和sortOrder排序的查询方法
- 查询行程时按sortOrder排序返回，确保前端显示顺序一致

#### 6.2 前端实现

**地图点位联动：**
- 为每个Leaflet地图点位绑定点击事件，传递行程ID
- 行程列表项绑定点击事件，触发地图点位高亮和居中
- 使用CSS类名控制高亮样式（背景色、边框、阴影等）
- 使用ref引用和scrollIntoView实现滚动至可视区域
- 维护高亮状态变量，支持单选和取消选择

**行程拖拽排序：**
- 集成vuedraggable库（基于Sortable.js）
- 配置拖拽选项：handle（拖拽手柄）、animation（动画时延）、ghostClass（占位符样式）
- 拖拽结束后触发onEnd事件，更新本地数据顺序
- 调用排序接口同步后端数据，提供加载状态反馈
- 拖拽手柄使用图标（如：⋮⋮或≡）提升可发现性

#### 6.3 交互流程

**地图点位联动（点击地图→行程）：**
1. 用户点击地图上的某个点位标记
2. 前端获取点位对应的行程ID（通过marker绑定的数据）
3. 清除之前的高亮状态
4. 左侧行程列表中对应项添加高亮样式（highlighted类名）
5. 使用scrollIntoView({ behavior: 'smooth', block: 'center' })自动滚动
6. 地图点位同时高亮（改变图标颜色或添加脉冲动画）

**地图点位联动（点击行程→地图）：**
1. 用户点击左侧行程列表中的某个行程项
2. 前端获取行程对应的经纬度坐标
3. 清除之前的高亮状态
4. 行程项添加高亮样式
5. 地图平移至该坐标并设置合适的缩放级别（map.setView）
6. 对应点位标记高亮显示（改变图标样式）

**行程拖拽排序：**
1. 用户按住行程项的拖拽手柄（⋮⋮图标）
2. 拖拽至目标位置，显示半透明占位符
3. 松开后行程列表顺序立即更新（前端乐观更新）
4. 显示加载状态，调用PUT /api/dailyPlan/sort/{planId}接口
5. 接口返回成功后，清除加载状态
6. 接口返回失败时，回滚前端顺序并提示用户

### 7. 交付标准

- 点击地图点位，对应行程项正确高亮并滚动至可视区域
- 点击行程项，对应地图点位正确高亮并居中显示
- 点击空白区域，所有高亮状态正确取消
- 行程拖拽排序功能正常，顺序正确保存
- 拖拽过程中视觉反馈流畅（占位符、动画效果）
- 排序后数据一致性正确（前后端同步，刷新后顺序保持）
- 高亮样式明显可辨识（背景色、边框、阴影）
- 无控制台报错，用户体验流畅
- 排序接口响应时间<500ms
