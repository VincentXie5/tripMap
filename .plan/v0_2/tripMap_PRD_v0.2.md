### 1. 版本目标

在v0.1核心功能闭环的基础上，补全旅行计划和每日行程的编辑、删除操作能力，实现完整的CRUD（增删改查）功能闭环，提升用户体验和数据管理灵活性。

### 2. 版本范围

#### 2.1 必做核心功能

**旅行计划管理增强：**
- 编辑旅行计划：支持修改计划标题、开始日期、结束日期
- 删除旅行计划：支持删除已有旅行计划（级联删除关联的每日行程）
- 计划列表操作入口：每个计划项提供编辑、删除操作按钮

**每日行程管理增强：**
- 编辑每日行程：支持修改行程时段、行程地点
- 删除每日行程：支持删除单条行程记录
- 行程列表操作入口：每个行程项提供编辑、删除操作按钮

**交互优化：**
- 编辑操作采用弹窗/抽屉表单，复用新建表单组件
- 删除操作需二次确认提示，防止误删
- 操作完成后自动刷新列表数据
- 地图标记点位随数据变更实时更新

#### 2.2 本版本暂不实现功能

- 用户登录、注册、权限管理
- 地图点位与行程的点击联动（点击地图点位高亮对应行程）
- 响应式适配移动端、平板
- 行程导出、分享、路线规划
- 行程拖拽排序
- 批量删除操作
- 操作历史记录与撤销功能

### 3. 页面结构设计

```Plain Text
首页（全屏布局）
├─ 左侧区域（固定占比40%）
│  ├─ 新建旅行计划表单
│  ├─ 旅行计划列表（日期区间 + 计划名 + [编辑] [删除]按钮）
│  │   └─ 编辑计划弹窗（复用表单，预填数据）

└─ 右侧区域（固定占比60%）
   └─ Leaflet地图容器（展示所有行程地点标记，数据变更实时刷新）
```

### 4. 数据模型设计

#### 4.1 旅行计划实体（TravelPlan）

保持v0.1设计不变：
- id：主键ID，Long类型，自增
- title：计划名称，字符串类型
- startDate：旅行开始日期，日期类型
- endDate：旅行结束日期，日期类型

#### 4.2 每日行程实体（DailyPlan）

保持v0.1设计不变：
- id：主键ID，Long类型，自增
- planId：关联旅行计划ID，Long类型
- planDate：行程当天日期，日期类型
- time：行程具体时段，字符串类型（如：08:00、14:00）
- location：行程地点，字符串类型（如：北京故宫、西安兵马俑）

### 5. 核心接口清单

#### 5.1 v0.1已有接口（复用）

- POST /api/travelPlan：新建旅行计划
- GET /api/travelPlan：查询全部旅行计划列表
- POST /api/dailyPlan：新增每日行程
- GET /api/dailyPlan/{planId}：根据计划ID查询对应全部行程

#### 5.2 v0.2新增接口

**旅行计划接口：**
- PUT /api/travelPlan/{id}：更新旅行计划
  - 请求体：title、startDate、endDate
  - 响应：更新后的旅行计划对象
  
- DELETE /api/travelPlan/{id}：删除旅行计划
  - 级联删除关联的所有每日行程
  - 响应：成功状态

**每日行程接口：**
- PUT /api/dailyPlan/{id}：更新每日行程
  - 请求体：planDate、time、location
  - 响应：更新后的每日行程对象
  
- DELETE /api/dailyPlan/{id}：删除每日行程
  - 响应：成功状态

### 6. 技术实现要点

#### 6.1 后端实现

- TravelPlanController：新增 updateTravelPlan、deleteTravelPlan 方法
- DailyPlanController：新增 updateDailyPlan、deleteDailyPlan 方法
- Service层：实现业务逻辑，删除计划时级联删除行程
- Repository层：使用JPA内置的deleteById方法

#### 6.2 前端实现

- PlanList组件：为每个计划项添加编辑、删除按钮
- DailyPlanList组件：为每个行程项添加编辑、删除按钮
- 复用 TravelPlanForm 组件作为编辑弹窗表单
- 复用 DailyPlanForm 组件作为编辑弹窗表单
- 使用 Element Plus 的 Dialog 组件实现弹窗
- 使用 Element Plus 的 MessageBox 实现删除确认
- 调用更新后的 travelApi.ts 中的接口方法

#### 6.3 交互流程

**编辑旅行计划：**
1. 点击计划项的[编辑]按钮
2. 弹出编辑弹窗，表单预填当前计划数据
3. 修改表单内容，点击确认
4. 调用更新接口，成功后关闭弹窗并刷新列表
5. 地图标记点位自动更新

**删除旅行计划：**
1. 点击计划项的[删除]按钮
2. 弹出确认对话框："确定删除该旅行计划吗？关联的行程数据也将被删除。"
3. 点击确认后调用删除接口
4. 成功后刷新计划列表，清空行程列表
5. 地图标记点位自动清空

**编辑每日行程：**
1. 点击行程项的[编辑]按钮
2. 弹出编辑弹窗，表单预填当前行程数据
3. 修改表单内容，点击确认
4. 调用更新接口，成功后关闭弹窗并刷新行程列表
5. 地图标记点位自动更新

**删除每日行程：**
1. 点击行程项的[删除]按钮
2. 弹出确认对话框："确定删除该行程吗？"
3. 点击确认后调用删除接口
4. 成功后刷新行程列表
5. 地图标记点位自动更新
