### 1. 版本目标

完成前后端项目从零搭建、核心功能闭环，实现可本地运行、可演示的基础版本，只保留核心必备功能，砍掉所有非必要模块，快速落地可用Demo。

### 2. 版本范围

#### 2.1 必做核心功能

- 后端：SpringBoot项目初始化、H2数据库配置、基础实体类、增删查接口开发

- 前端：Vue3项目初始化、左右分栏基础页面搭建、Element Plus引入、接口对接

- 旅行计划：新建计划、展示全部计划列表（展示字段：开始日期-结束日期、计划标题）

- 每日行程：为对应计划添加行程、展示行程列表（展示字段：时段：地点）

- 地图联动：加载Leaflet + OpenStreetMap，自动渲染行程内所有地点的标记点位

- 前后端联调：解决跨域问题，接口正常调用，数据正常渲染

#### 2.2 本版本暂不实现功能

- 用户登录、注册、权限管理

- 旅行计划、行程的编辑、删除操作

- 地图点位与行程的点击联动

- 响应式适配移动端、平板

- 行程导出、分享、路线规划

- 复杂样式美化、交互动画

### 3. 页面结构设计

```Plain Text
首页（全屏布局）
├─ 左侧区域（固定占比40%）
│  ├─ 新建旅行计划表单
│  ├─ 旅行计划列表（日期区间+计划名）
│  └─ 选中计划的每日行程列表（时间：地点）
└─ 右侧区域（固定占比60%）
   └─ Leaflet地图容器（展示所有行程地点标记）

```

### 4. 数据模型设计

#### 4.1 旅行计划实体（TravelPlan）

- id：主键ID，Long类型，自增

- title：计划名称，字符串类型

- startDate：旅行开始日期，日期类型

- endDate：旅行结束日期，日期类型

#### 4.2 每日行程实体（DailyPlan）

- id：主键ID，Long类型，自增

- planId：关联旅行计划ID，Long类型

- planDate：行程当天日期，日期类型

- time：行程具体时段，字符串类型（如：08:00、14:00）

- location：行程地点，字符串类型（如：北京故宫、西安兵马俑）

### 5. 核心接口清单

- POST /api/travelPlan：新建旅行计划

- GET /api/travelPlan：查询全部旅行计划列表

- POST /api/dailyPlan：新增每日行程

- GET /api/dailyPlan/{planId}：根据计划ID查询对应全部行程

### 6. 交付标准

- 前后端项目可独立启动，无报错

- 新建计划、添加行程功能正常，数据可持久化

- 左侧列表正常渲染计划和行程数据

- 右侧地图正常加载，行程地点可渲染标记

- 无跨域、参数报错等联调问题
